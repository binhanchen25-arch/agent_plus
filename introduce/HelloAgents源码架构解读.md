# HelloAgents 源码架构解读

> 基于 Datawhale Hello-Agents 教程的多智能体框架，采用 OpenAI 原生 API 构建，提供灵活可扩展的智能体开发体验。

**定位**：多智能体框架

## 三、核心模块详解

### 3.1 核心层（core）

#### Agent 基类（`core/agent.py`）

所有 Agent 的抽象基类，定义统一接口：

- **`run(input_text, **kwargs) -> str`**：抽象方法，子类必须实现
- **`add_message(message)`**：追加消息到历史
- **`clear_history()`**：清空历史
- **`get_history()`**：获取历史副本

依赖 `HelloAgentsLLM`、`Message`、`Config`，通过 `_history` 维护对话上下文。

#### HelloAgentsLLM（`core/llm.py`）

统一 LLM 客户端，面向 OpenAI 兼容 API：

- **多提供商支持**：OpenAI、DeepSeek、Qwen、ModelScope、Kimi、智谱、Ollama、vLLM、本地等
- **自动检测**：根据环境变量、API Key 格式、base_url 推断 provider
- **主要方法**：
  - `think(messages, temperature)`：流式调用，逐块返回
  - `invoke(messages, **kwargs)`：非流式调用，返回完整文本
  - `stream_invoke()`：`think` 的别名

设计原则：参数优先、环境变量兜底、默认流式输出。

#### Message（`core/message.py`）

Pydantic 消息模型，包含 `content`、`role`、`timestamp`、`metadata` 等字段。

---

### 3.2 Agent 实现层（agents）

| Agent | 说明 | 典型用法 |
|-------|------|----------|
| **SimpleAgent** | 纯对话，无工具 | 简单问答 |
| **FunctionCallAgent** | 使用 OpenAI Function Calling | 工具调用、结构化输出 |
| **ReActAgent** | Thought → Action → Observation 循环 | 需要推理与工具交替的任务 |
| **ReflectionAgent** | 带自我反思 | 需要纠错与改进 |
| **PlanAndSolveAgent** | 先规划再执行 | 多步骤任务 |
| **ToolAwareSimpleAgent** | 工具感知的简单 Agent | 轻量工具使用 |
| **SkillAgent** | 继承 FunctionCallAgent，加载技能目录 | 技能驱动、按需加载 |

#### FunctionCallAgent 核心流程

1. 构建系统提示，注入工具描述
2. 从 `ToolRegistry` 生成 OpenAI function schema
3. 调用 `client.chat.completions.create(tools=...)`
4. 解析 `tool_calls`，执行 `ToolRegistry.execute_tool()`
5. 将工具结果以 `role: tool` 消息追加，继续调用 LLM
6. 达到 `max_tool_iterations` 或 LLM 不再调用工具时结束

#### ReActAgent 核心流程

1. 使用固定提示模板，要求输出 `Thought:` 和 `Action:` 格式
2. 解析 `Action: tool_name[args]` 或 `Finish[结论]`
3. 调用工具得到 `Observation`
4. 将 Thought、Action、Observation 追加到历史，进入下一轮
5. 直到 `Finish` 或达到 `max_steps`

---

### 3.3 工具系统（tools）

#### Tool 基类（`tools/base.py`）

- **抽象方法**：`run(parameters) -> str`、`get_parameters() -> List[ToolParameter]`
- **`to_openai_schema()`**：生成 OpenAI function calling 所需 schema
- **可展开工具**：`expandable=True` 时，可用 `@tool_action` 将方法拆成多个子工具
- **AutoGeneratedTool**：从方法签名和 docstring 自动解析参数

#### ToolRegistry（`tools/registry.py`）

- **register_tool(tool)**：注册 Tool 对象，支持自动展开
- **register_function(name, description, func)**：直接注册函数
- **execute_tool(name, input_text)**：执行工具并返回字符串
- **get_tools_description()**：生成供 LLM 阅读的工具说明

#### 工具链与异步执行

- **ToolChain / ToolChainManager**：顺序执行，支持变量替换
- **AsyncToolExecutor**：并行、批量执行工具

---

### 3.4 技能系统（skills）

基于 Anthropic Agent Skills 协议，将技能目录暴露为可调用工具。

#### Skill 数据模型（`skills/base.py`）

```python
@dataclass
class Skill:
    name: str           # 技能名称
    description: str   # 描述（供 LLM 发现）
    instructions: str  # SKILL.md 正文
    path: Path         # 技能目录
    references: List[str]
    resources: List[str]
    scripts: List[str]
    metadata: Dict
```

#### SkillLoader（`skills/loader.py`）

- 从目录加载 `SKILL.md`
- 解析 YAML 前置元数据与 Markdown 正文
- `load_resource(skill, path)`：按需加载 reference、resources 等

#### SkillTool（`skills/skill_tool.py`）

- 继承 `Tool`，将 `Skill` 暴露给 FunctionCallAgent
- **渐进式披露**：LLM 先看到 name/description，发起 function call 时再返回完整 instructions
- 可选参数：`context`、`load_resources`，用于按需加载附属资源

#### SkillAgent

- 从指定目录加载技能，注册为 `SkillTool`
- 继承 `FunctionCallAgent`，复用函数调用流程

---

### 3.5 记忆系统（memory）

#### 基础组件（`memory/base.py`）

- **MemoryItem**：id、content、memory_type、user_id、timestamp、importance、metadata
- **MemoryConfig**：存储路径、容量、重要性阈值、衰减因子等
- **BaseMemory**：抽象接口，定义 `add`、`retrieve`、`update`、`remove`、`clear`、`get_stats`

#### 记忆类型（`memory/types/`）

- **WorkingMemory**：短期工作记忆，容量与 TTL 受限
- **EpisodicMemory**：情景记忆，按时间线组织
- **SemanticMemory**：语义记忆，向量检索
- **PerceptualMemory**：感知记忆，支持多模态

#### MemoryManager

协调多种记忆类型，提供统一检索与更新接口。

---

### 3.6 上下文构建（context）

#### ContextBuilder（`context/builder.py`）

实现 **GSSC 流水线**：

1. **Gather**：从历史、记忆、RAG、工具结果等多源收集候选信息
2. **Select**：按相关性、新近性、多样性筛选，在 token 预算内选择
3. **Structure**：组织成 [Role & Policies]、[Task]、[State]、[Evidence]、[Context]、[Output] 等区块
4. **Compress**：超预算时截断或压缩

依赖 `MemoryTool`、`RAGTool`，可与 Agent 配合构建高质量上下文。

## 五、设计模式与扩展点

| 模式 | 说明 |
|------|------|
| **OpenAI 兼容** | `Tool.to_openai_schema()` 适配 function calling |
| **渐进式披露** | SkillTool 先暴露 name/description，调用时再返回完整 instructions |
| **可展开工具** | `@tool_action` + `expandable=True` 支持多 action 工具 |
| **Provider 抽象** | HelloAgentsLLM 统一多 LLM 提供商 |
| **分层记忆** | Working / Episodic / Semantic / Perceptual 分工明确 |
| **GSSC 流水线** | 上下文构建标准化为 Gather-Select-Structure-Compress |

**扩展建议**：

- 新增 Agent：继承 `Agent`，实现 `run()`
- 新增工具：继承 `Tool`，实现 `run()` 和 `get_parameters()`
- 新增技能：在目录下创建 `SKILL.md`，由 SkillLoader 加载
- 新增记忆类型：继承 `BaseMemory`，实现抽象方法

## 七、快速上手示例

```python
from hello_agents import HelloAgentsLLM, FunctionCallAgent, ToolRegistry
from hello_agents.tools.builtin import SearchTool, CalculatorTool

# 1. 初始化 LLM
llm = HelloAgentsLLM(model="gpt-4", provider="openai")

# 2. 注册工具
registry = ToolRegistry()
registry.register_tool(SearchTool())
registry.register_tool(CalculatorTool())

# 3. 创建 Agent
agent = FunctionCallAgent(name="助手", llm=llm, tool_registry=registry)

# 4. 运行
response = agent.run("北京今天天气怎么样？")
print(response)
```
